# Switch

<Tagline>A switch offers an on or off choice to a user.</Tagline>

<ComponentInfo
    usage={"import { Switch } from \"@orbit-ui/react-components\""}
    ariaPath="#checkbox"
    githubPath="/packages/react-components/src/switch/src"
/>

## Usage

### Label

A default switch.

<Canvas>
    <Story name="default">
        <Switch>Engines</Switch>
    </Story>
</Canvas>

### Checked

A switch can be checked.

<Canvas>
    <Story name="checked">
        <Switch defaultChecked>Engines</Switch>
    </Story>
</Canvas>


### No label

A switch can be rendered without a label.

<Canvas>
    <Story name="no label">
        <Switch />
    </Story>
</Canvas>

### Disabled

A switch can be disabled.

<Canvas>
    <Story name="disabled">
        <Switch disabled>Engines</Switch>
    </Story>
</Canvas>

### Icon

A switch can have [icons](?path=/docs/icon-gallery--page) after his text.

<Canvas>
    <Story name="icon">
        <Switch>
            <Text>Pressurization</Text>
            <LightbulbIcon />
        </Switch>
    </Story>
</Canvas>

### Counter

A switch can have a [counter](?path=/docs/counter--default-story) after his text.

<Canvas>
    <Story name="counter">
        <Inline>
            <Switch>
                <Text>Pressurization</Text>
                <Counter>6</Counter>
            </Switch>
            <Switch>
                <Text>Engines</Text>
                <Counter variant="divider">6</Counter>
            </Switch>
        </Inline>
    </Story>
</Canvas>

### Size

A switch can vary in size.

<Canvas>
    <Story name="size">
        <Inline verticalAlign="center">
            <Switch size="sm">Engines</Switch>
            <Switch>Engines</Switch>
        </Inline>
    </Story>
</Canvas>

### Reverse

A checkbox checkmark can be to the left or to the right of his label.

<Canvas>
    <Story name="reverse">
        <Switch reverse>
            <InfoIcon />
            <Text>Engines</Text>
            <Counter variant="divider">6</Counter>
        </Switch>
    </Story>
</Canvas>

### Validation

A switch can display a validation state to communicate to the user whether the current value is valid or invalid. Implement your own validation logic in your app and pass either `"valid"` or `"invalid"` to the input via the `validationState` prop.

The example below illustrates how one would validate if the user has checked the switch.

<FileCanvas filePath="/switch/docs/CheckedValidation">
    <CheckedValidation />
</FileCanvas>

### Render Props

A switch can update is content based on the `isChecked` value by providing a render prop function.

<Canvas>
    <Story name="render props">
        <Switch>
            {
                ({ isChecked }) => {
                    return isChecked ? "On" : "Off"
                }
            }
        </Switch>
    </Story>
</Canvas>

### Controlled

A switch can handle `checked` state in controlled mode.

<FileCanvas filePath="/switch/docs/ControlledSwitch">
    <ControlledSwitch />
</FileCanvas>
