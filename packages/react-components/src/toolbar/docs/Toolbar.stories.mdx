# Toolbar

<Tagline>A toolbar is a container for grouping a set of controls, such as buttons, menubuttons, or checkboxes.</Tagline>

## Usage

### Default

A default toolbar.

<Canvas>
    <Story name="default">
        <Toolbar>
            <TextInput placeholder="Destination" />
            <RadioGroup>
                <Radio value="roscosmos">Roscosmos</Radio>
                <Radio value="nasa">NASA</Radio>
            </RadioGroup>
            <CheckboxGroup>
                <ToggleButton variant="outline" value="round-trip">Round trip</ToggleButton>
                <ToggleButton variant="outline" value="one-way">One way</ToggleButton>
            </CheckboxGroup>
        </Toolbar>
    </Story>
</Canvas>

### Disabled

A toolbar control can be disabled.

<Canvas>
    <Story name="disabled control">
        <Toolbar>
            <TextInput placeholder="Destination" />
            <RadioGroup disabled>
                <Radio value="roscosmos">Roscosmos</Radio>
                <Radio value="nasa">NASA</Radio>
            </RadioGroup>
            <CheckboxGroup>
                <ToggleButton variant="outline" value="round-trip">Round trip</ToggleButton>

            </CheckboxGroup>
        </Toolbar>
    </Story>
</Canvas>

Or all the controls.

### Custom Components

Any component can be added to a toolbar and behave as an element of the group. However to benefit from state propagation features, a component must implement the `ToolbarContext`. This is usually done by using the `useToolbarProps` hook.
