# IconLink

Links allow a user to navigate between pages, scroll to a have page section, access files, or even to handle email addresses.

## Content

An icon link accept an icon element as children.

### Accessibility

An icon link doesn't have a label, an accessible name must be provided through `aria-label` prop. See [WCAG practices](https://www.w3.org/TR/WCAG20-TECHS/ARIA6.html).

## Rendering

### Button

An icon link can be rendered as a button.

<Preview>
    <Story name="button">
        <IconLink as="button" aria-label="Add" href="#">
            <AddIcon />
        </IconLink>
    </Story>
</Preview>

### React Router

To integrate with a routing library like [React Router](https://reactrouter.com) a text link can be rendered as a [React Router Link](https://reactrouter.com/web/api/Link) or as any custom component.

<Preview>
    <Story name="react router link">
        <IconLink as={RouterLink} to="/flight-details" aria-label="Add">
            <AddIcon />
        </IconLink>
    </Story>
</Preview>
