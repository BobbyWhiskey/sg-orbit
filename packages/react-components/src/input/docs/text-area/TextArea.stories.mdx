# TextArea

<Tagline>A text area is a multi-line plain-text editing control, useful when you want to allow users to enter a sizeable amount of free-form text.</Tagline>

<ComponentInfo
    usage={"import { TextArea } from \"@orbit-ui/react-components\""}
    githubPath="/packages/react-components/src/input/src"
/>

## Usage

### Default

A default text input.

<Canvas>
    <Story name="default">
        <TextArea placeholder="Why should you go to space?" />
    </Story>
</Canvas>

### Value

A text area can have a value.

<Canvas>
    <Story name="value">
        <TextArea defaultValue="To fly high!" placeholder="Why should you go to space?" />
    </Story>
</Canvas>

### Button

A text area can contain a [button](?path=/docs/button--default-story).

<Canvas>
    <Story name="button">
        <Inline>
            <TextArea button={<CrossButton aria-label="Clear value" />} placeholder="Why should you go to space?" />
        </Inline>
    </Story>
</Canvas>

### Max Rows

A text are can be limited to a maximum number of rows.

<Canvas>
    <Story name="max rows">
        <Inline>
            <TextArea maxRows={5} placeholder="Why should you go to space?" />
        </Inline>
    </Story>
</Canvas>

### Disabled

A text area can be disabled.

<Canvas>
    <Story name="disabled">
        <TextArea disabled placeholder="Why should you go to space?" />
    </Story>
</Canvas>

### Readonly

A text area can be readonly.

<Canvas>
    <Story name="readonly">
        <TextArea readOnly defaultValue="NASA selected SpaceX to develop a lunar optimized Starship to transport crew between lunar orbit and the surface of the Moon as part of NASAâ€™s Artemis program." />
    </Story>
</Canvas>

### Loading

A text area can show that it is currently loading data.

<Canvas>
    <Story name="loading">
        <TextArea loading />
    </Story>
</Canvas>

### Fluid

A text area can take the width of its container.

<Canvas>
    <Story name="fluid">
        <TextArea fluid placeholder="Why should you go to space?" />
    </Story>
</Canvas>

### Validation

A text input can display a validation state to communicate to the user whether the current value is valid or invalid. Implement your own validation logic in your app and pass either `"valid"` or `"invalid"` to the input via the `validationState` prop.

The example below illustrates how one would validate that a text area doesn't exceed 25 characters (yes it could also be done with maxLength).

<FileCanvas filePath="/input/docs/text-area/LengthValidation">
    <LengthValidation />
</FileCanvas>

### Transparent

A text area can be transparent.

<Canvas>
    <Story name="transparent">
        <TextArea variant="transparent" placeholder="Why should you go to space?" />
    </Story>
</Canvas>
