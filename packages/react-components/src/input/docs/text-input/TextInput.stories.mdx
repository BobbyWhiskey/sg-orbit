# TextInput

<Tagline>A text input allow a user to enter and edit a text.</Tagline>

    usage={"import { TextArea } from \"@orbit-ui/react-components\""}

## Usage

### Default

A default text input.

<Canvas>
    <Story name="default">
        <TextInput placeholder="Where to?" />
    </Story>
</Canvas>

### Value

A text input can have a value.

<Canvas>
    <Story name="value">
        <TextInput defaultValue="Deep space" placeholder="Where to?" />
    </Story>
</Canvas>

### Icon

A text input can contain an [icon](?path=/docs/icon-gallery--page).

<Canvas>
    <Story name="icon">
        <TextInput icon={<EditIcon />} placeholder="Where to?" />
    </Story>
</Canvas>

### Button

A text input can contain a [button](?path=/docs/button--default-story).

<Canvas>
    <Story name="button">
        <TextInput button={<CrossButton aria-label="Clear value" />} placeholder="Where to?" />
    </Story>
</Canvas>

### Disabled

A text input can be disabled.

<Canvas>
    <Story name="disabled">
        <TextInput disabled placeholder="Where to?" />
    </Story>
</Canvas>

### Readonly

A text input can be readonly.

<Canvas>
    <Story name="readonly">
        <TextInput readOnly defaultValue="Deep space" />
    </Story>
</Canvas>

### Loading

A text input can show that it is currently loading data.

<Canvas>
    <Story name="loading">
        <TextInput loading />
    </Story>
</Canvas>

### Fluid

A text input can take the width of its container.

<Canvas>
    <Story name="fluid">
        <TextInput fluid placeholder="Where to?" />
    </Story>
</Canvas>

### Validation

A text input can display a validation state to communicate to the user whether the current value is valid or invalid. Implement your own validation logic in your app and pass either `"valid"` or `"invalid"` to the input via the `validationState` prop.

The example below illustrates how one would validate if the user has entered a valid email.

<FileCanvas filePath="/input/docs/text-input/EmailValidation">
    <EmailValidation />
</FileCanvas>

### Transparent

A text input can be transparent.

<Canvas>
    <Story name="transparent">
        <TextInput variant="transparent" placeholder="Where to?" />
    </Story>
</Canvas>

## PasswordInput

A specialized text input which show / hide a password.

    <Story name="password">
        <PasswordInput />
    </Story>
</Canvas>
